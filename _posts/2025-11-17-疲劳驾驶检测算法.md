# 驾驶员疲劳检测系统 - 核心算法介绍

## 算法概述

驾驶员疲劳检测系统基于计算机视觉和机器学习技术，通过分析面部特征实时监测驾驶员疲劳状态。系统采用多维度特征融合策略，结合眼部、嘴部和眨眼行为特征，实现高精度的疲劳检测。

## 核心算法组成

### 1. 面部特征点检测
- 使用MediaPipe Face Mesh提取468个面部关键点
- 重点关注眼部(32个点)和嘴部(20个点)特征
- 实时跟踪面部特征点位置变化

### 2. 眼部疲劳检测
- **眼睛纵横比(EAR)算法**：
  ```
  EAR = (|p2-p6| + |p3-p5|) / (2|p1-p4|)
  ```
- 通过EAR值变化判断眼睛开合状态
- 结合连续闭眼时间阈值(6秒)识别疲劳

### 3. 嘴部疲劳检测
- **嘴巴开合度算法**：
  ```
  Mouth_Ratio = 嘴巴高度 / 嘴巴宽度
  ```
- 检测哈欠行为(嘴巴开合度 > 阈值)
- 识别疲劳相关的嘴部特征

### 4. 眨眼频率分析
- 统计单位时间内的眨眼次数
- 正常眨眼频率：15-20次/分钟
- 疲劳阈值：< 7次/分钟
- 通过眨眼频率变化判断疲劳程度

### 5. 综合疲劳评估
- **滑动窗口疲劳率计算**：
  ```
  疲劳率 = (时间窗口内疲劳帧数 / 总帧数) × 100%
  ```
- **置信度计算**：
  ```
  置信度 = min(1.0, 疲劳率 / 阈值)
  ```
- 多特征融合决策，提高检测准确性

## 算法特点

1. **多维度特征融合**：结合眼部、嘴部和眨眼行为特征，提高检测准确性
2. **时间序列分析**：通过滑动窗口机制分析连续帧状态，避免瞬时误判
3. **自适应阈值**：提供多种灵敏度级别，适应不同个体和环境
4. **实时性能优化**：帧率控制和高效计算，确保实时检测流畅性

## 技术指标

- 检测延迟：< 100ms
- 实时帧率：10-15 FPS
- 准确率：> 90%
- 支持分辨率：640×480(推荐)

该算法通过多维度特征分析和时间序列建模，实现了对驾驶员疲劳状态的准确、实时检测，为驾驶安全提供有效保障。
